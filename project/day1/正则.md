正则：

```python
学习方法： --》 从案列入手

核心：
	编写处理网页文本的程序时，经常会有查找符合某些复杂规则的字符串的需要。
正则表达式就是用于描述这些规则的工具

组成：
	由普通字符（例如字符 a 到z）以及特殊字符（称为"元字符"）组成的文字模式
    
工作模式：
	模式描述在搜索文本时要匹配的一个或多个字符串。正则表达式作为一个模板，将某个字符模式与所搜索的字符串进行匹配
```

入门：

```python
英文文献中所有的 we 单词 ?

注意：
	正则表达式工具一般可以设置为忽略大小写
    
匹配：  \bwe\b    

说明：
	\b  :  特殊代码，被称为元字符，代表着单词的开头或结尾，也就是单词的分界处，
            它不代表着英语中空格、标点符号、换行等单词分隔符，只是用来匹配一个位置
            
```



```python
把 we、work 和它们之间的所有内容都匹配出来 ??

匹配： \bwe\b.*\bwork\b。

说明：
	. 这个元字符的含义是匹配除了换行符的任意字符
    * 元字符不是代表着字符，而是代表着数量，含义是 * 前面的内容可以连续重复任意次使得整个
表达式被匹配。
	.*整体的意思就非常明显了，代表着可以匹配任意数量不换行的字符
    
注释：
	\bwe\b.*\bwork\b 作用就是先匹配出 we 单词，接着在匹配任意的字符(非换行)，直到匹配到 work 单词结束
	
```

常用元字符：

```python
.   匹配除换行符以外的任意字符
\b  匹配单词的开始或结束
\d  匹配数字
\w  匹配字母或数字或下划线或汉字
\s  匹配任意空白符，包括 空格，制表符(Tab)，换行符，中文全角空格等
^   匹配字符串的开始
$   匹配字符串的结束


分类：
	数字， 字母 ， 字符串 ， 换行符， 空白符，开始 /结束 ， 数量
```

重复：

```python
将匹配重复的限定符（指定数量的代码）?

*      重复零次或更多次
+      重复一次或更多次
?      重复零次或一次
{n}    重复 n 次
{n,}   重复 n 次或更多次
{n,m}  重复 n 到 m 次

eg:
    hello\d+ :匹配 hello 后面跟 1 个或更多数字
    ^\d{5,12}$ :匹配 5 到 12 个数字的字符串，例如 QQ 号
   we\d? : 匹配 we 后面跟 0 个或者一个数字，例如 we、we0 
```

字符集合：

```python
	匹配没有预定义元字符的字符集合，例如匹配a、b、c、d 和 e 中任意一个字符，这时候就需要自定义字符集合??
    
    正则表达式是通过 [ ]来实现，[abcde] 就是匹配 abcde 中的任意一个字符，[.?!] 匹配标点符号(.或?或!)
    
    [0-9]代表的含义与\d 是完全一致的，代表一位数字；[a-z0-9A-Z_]也完全等同于\w（只考虑英文），代表着 26 个字母中的大小写、0-9 数字和下划线中的任一个字符
```

贪婪与懒惰：

```python
*?       重复任意次，但尽可能少重复
+?       重复 1 次或更多次，但尽可能少重复
??       重复 0 次或 1 次，但尽可能少重复
{n,m}?   重复 n 到 m 次，但尽可能少重复
{n,}?    重复 n 次以上，但尽可能少重复
```

处理选项：

```python
忽略大小写          匹配时不区分大小写。
多行模式           更改^和$的含义，使它们分别在任意一行的行首和行尾匹配，
			而不仅仅在整个字符串的开头和结尾匹配。(在此模式下，$的精
			确含意是：匹配\n 之前的位置以及字符串结束前的位置。)
单行模式 			更改.的含义，使它与每一个字符匹配（包括换行符\n）。
忽略空白 			忽略表达式中的非转义空白并启用由#标记的注释。
显式捕获 			仅捕获已被显式命名的组
```
